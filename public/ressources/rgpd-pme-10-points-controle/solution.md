# Corrigé de l’exercice — RGPD en PME

## Exemple complet pour 3 outils d’une PME de 8 salariés

| Outil | Données personnelles | Accès | Conservation | Base légale | Action prioritaire |
|---|---|---|---|---|---|
| **CRM** | Nom, email, téléphone, adresse, historique d’achats | Commercial + dirigeant | 3 ans après dernier contact | Contrat / intérêt légitime | Auditer la base, supprimer les contacts inactifs |
| **Logiciel de paie** | RIB, salaire, contrat, adresse | Comptable externe + responsable RH | 5 ans après départ | Obligation légale | Restreindre les accès, échanges via portail sécurisé |
| **Drive partagé** | Devis, factures, CV, contrats | Tous les salariés | Variable, souvent indéfinie | Contrat / obligation légale | Dossiers par équipe avec droits restreints |

## Réponses aux questions de réflexion

### 1. Pouvez-vous supprimer facilement les données d’un ancien ?

Si la réponse est non, votre priorité est de :
- Identifier où ces données sont stockées.
- Créer une procédure de suppression standard.
- Désigner une personne en charge des demandes.

### 2. Vos salariés connaissent-ils les règles sur l’IA ?

Si ce n’est pas le cas :
- Rédigez une charte d’usage interne simple.
- Interdisez explicitement le collage de données personnelles dans les IA en ligne.
- Proposez une alternative locale (Ollama) ou une solution hébergée en France.

### 3. Avez-vous une procédure en cas de fuite ?

Créez une fiche réflexe avec :
- Qui alerter immédiatement.
- Quelles données sont concernées et combien de personnes.
- Comment notifier la CNIL si nécessaire.
- Comment informer les personnes concernées.
